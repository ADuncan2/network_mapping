summary.csv — Per-substation network mapping statistics
========================================================
Generated by: map_substation() in gridstock/mapfuncs.py
One row per mapped substation.

Column                  Description
------                  -----------

-- Location --
substation_fid          Integer FID of the substation (General Boundary or PMT Transformer row in lv_assets)
lat_lon                 Centroid of the substation geometry in WGS84, formatted as "lat, lon" for pasting into Google Maps.
                        Converted from EPSG:27700 via pyproj.

-- Substation properties --
is_pmt                  True if the substation is a pole-mounted transformer (Asset_Type = "PMT Transformer"),
                        False if ground-mounted (Asset_Type = "General Boundary").
rating_kva              Transformer rating in kVA (e.g. "500 kVA"). Populated by enrich_substations() during
                        preprocessing via spatial join of Transformer/PMT Transformer assets to the substation boundary.
                        May be semicolon-separated if multiple transformers matched.
infeed_voltage          Primary voltage feeding the substation (e.g. "11kV", "6.6kV"). Source: lv_assets column 18.

-- Network size --
nodes                   Total number of nodes discovered by DFS (excludes the substation node itself).
edges                   Total number of edges discovered by DFS.
service_points          Count of nodes with Asset_Type = "Service Point".
switches                Count of nodes with Asset_Type = "Switch".
cable_length_km         Total length of all edges in km (LV Conductor + LV Service + any other edge types).
conductor_length_km     Total length of edges with Asset_Type = "LV Conductor" in km. These are the main
                        distribution cables running along streets.
service_length_km       Total length of edges with Asset_Type = "LV Service" in km. These are the short
                        tails from the main cable to individual service points (premises).

-- DFS diagnostics --
dfs_calls               Total number of DFS function calls (node + edge visits).
dfs_time_s              Wall-clock time for the DFS phase in seconds.
dfs_max_depth           Maximum recursion depth reached during DFS.
depth_limit_hits        Number of times DFS hit the max recursion depth limit (2000) and had to backtrack.
hyper_edges             Number of hyper-edges encountered (edges with >2 terminals, e.g. tee-joints).
                        These are subdivided into regular sub-edges during DFS.
adjacent_substations    Number of times DFS encountered another substation and turned back. Indicates
                        how many neighbouring substations share edges with this network.

-- Validity flags --
warning_count           Number of warning-level log messages generated during mapping.
error_count             Number of error-level log messages generated during mapping.
budget_exceeded         True if the DFS was cut short because it exceeded 50,000 calls or 120 seconds.
                        Networks where this is True are likely incomplete.
disconnected_components Number of connected components in the mapped network graph. Ideally 1.
                        Values >1 indicate the network has disconnected segments.
orphan_nodes            Number of isolated nodes (degree 0) in the mapped network. Should be 0.
redundant_visits        Total count of nodes or edges that DFS visited more than once. Non-zero values
                        indicate the network has cycles or the DFS revisited already-mapped topology.
