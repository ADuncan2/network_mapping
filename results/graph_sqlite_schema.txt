graph.sqlite — Mapped network topology and asset properties
===========================================================
Generated by: NetworkData.write_to_db() in gridstock/recorder.py
              process_fid() in building_matching.py (building_matches table)
SQLite database using WAL journal mode for concurrent multiprocessing writes.

Table: incidence_list
---------------------
One row per edge in all mapped networks. Defines the network topology.

Column                  Description
------                  -----------
edge_fid                Integer FID of the edge (PRIMARY KEY). References edge_list.fid.
node_from               Integer FID of the node at one end of the edge. References node_list.fid.
node_to                 Integer FID of the node at the other end of the edge. References node_list.fid.
parent_substation       Integer FID of the substation this edge belongs to. Null for edges
                        discovered via switch expansion (see parent_switch).
parent_switch           Integer FID of the switch that "owns" this edge, if the edge was discovered
                        by expanding a switch rather than directly from a substation. Null otherwise.

Table: node_list
----------------
One row per node discovered by DFS across all mapped networks.

Column                  Description
------                  -----------
fid                     Integer FID of the node (PRIMARY KEY). From lv_assets.
Geometry                WKB blob of the node geometry (Point, EPSG:27700). Deserialise with
                        shapely.wkb.loads() or GeoDataFrame.from_wkb().
Asset_Type              Type of network asset (e.g. "Service Point", "Switch", "Joint",
                        "Disconnection Point", "PMT Transformer").
Voltage                 Nominal voltage in volts (e.g. 400).
Material                Cable/conductor material (e.g. "Copper", "Aluminium"). Null for most nodes.
Conductors_Per_Phase    Number of conductors per phase. Typically null for nodes.
Cable_Size              Cable cross-section size (e.g. "185", "95"). Typically null for nodes.
Insulation              Insulation type. Typically null for nodes.
Installation_Date       Date the asset was installed. May be null.
Phases_Connected        Phase connection code (e.g. "RYB", "R").
Sleeve_Type             Joint sleeve type, if applicable. Null for most nodes.
Associated_Cable        FID of an associated cable, if any. Null for most nodes.
Switch_Status           Status of a switch node (e.g. "Closed", "Open"). Null for non-switch nodes.

Table: edge_list
----------------
One row per edge discovered by DFS across all mapped networks.
Same schema as node_list.

Column                  Description
------                  -----------
fid                     Integer FID of the edge (PRIMARY KEY). From lv_assets.
Geometry                WKB blob of the edge geometry (LineString, EPSG:27700).
Asset_Type              Type of network asset (e.g. "LV Conductor", "LV Service").
Voltage                 Nominal voltage in volts (e.g. 400).
Material                Cable/conductor material (e.g. "Copper", "Aluminium").
Conductors_Per_Phase    Number of conductors per phase.
Cable_Size              Cable cross-section size (e.g. "185", "95").
Insulation              Insulation type (e.g. "XLPE", "Paper").
Installation_Date       Date the asset was installed. May be null.
Phases_Connected        Phase connection code (e.g. "RYB", "R").
Sleeve_Type             Typically null for edges.
Associated_Cable        FID of an associated cable, if any.
Switch_Status           Typically null for edges.

Table: mapped_substations
-------------------------
Tracks which substations have been successfully mapped. Used for resume support
and to avoid re-mapping substations on restart.

Column                  Description
------                  -----------
substation_fid          Integer FID of the mapped substation (PRIMARY KEY).

Table: mapped_switches
----------------------
Tracks which switches have been expanded during mapping.

Column                  Description
------                  -----------
switch_fid              Integer FID of the mapped switch (PRIMARY KEY).

Table: building_matches
-----------------------
One row per service point matched (or unmatched) to a building.
Populated by building_matching.py after network mapping is complete.

Column                  Description
------                  -----------
service_point_fid       Integer FID of the service point node (composite PRIMARY KEY with substation_fid).
substation_fid          Integer FID of the parent substation (composite PRIMARY KEY with service_point_fid).
toid                    OS MasterMap TOID of the matched building. Null if unmatched.
uprn                    Unique Property Reference Number of the matched building. May be null.
match_method            How the match was made: "overlap", "buffer_1m", or "unmatched".
dist_to_building        Distance in metres from the service point to the matched building.
                        Null for overlap matches and unmatched nodes.
activity_type           Land-use classification of the matched building (e.g. "domestic",
                        "non-domestic"). Null if unmatched.
