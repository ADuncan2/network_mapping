building_match_summary.csv — Per-substation building match summary
===================================================================
Generated by: process_fid() in building_matching.py
One row per substation that was processed for building matching.

Column                  Description
------                  -----------
fid                     Integer FID of the substation.
status                  Outcome of the building matching attempt:
                          "success" — matching completed normally.
                          "skipped" — network had no service points, so matching was not attempted.
                          "failed"  — an error occurred during matching (see "error" column).
time                    Wall-clock time in seconds for the full process_fid() call (load network,
                        find buildings, match, save). Only present for "success" status.
buildings_matched       Number of service points that were matched to a building (toid is not null).
buildings_unmatched     Number of service points with no building match (designated as lamp posts
                        or unresolved).
match_rate              Fraction of service points matched: buildings_matched / total service points.
                        Range 0.0–1.0.
pct_overlap             Fraction of matched service points that were matched via direct overlap
                        (vs buffer): overlap_count / buildings_matched. Range 0.0–1.0.
                        Higher values indicate better spatial alignment between service points
                        and building footprints.
error                   Error message string. Only present when status = "failed".
reason                  Reason for skipping. Only present when status = "skipped"
                        (e.g. "no service points").
