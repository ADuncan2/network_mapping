building_matches.csv — Per-service-point building match results
================================================================
Generated by: process_fid() in building_matching.py
One row per service point node in each mapped network.

Column                  Description
------                  -----------
substation_fid          Integer FID of the parent substation this service point belongs to.
service_point_fid       Integer FID of the service point node (from node_list in graph.sqlite).
toid                    OS MasterMap TOID of the matched building. Null if unmatched.
uprn                    Unique Property Reference Number of the matched building. May not always be present.
match_method            How the service point was matched to a building:
                          "overlap"   — the service point geometry intersects a building polygon directly.
                          "buffer_1m" — no direct overlap, but a 1m buffer around the service point
                                        intersected exactly one building. Multi-match cases are excluded.
                          "unmatched" — no building found. These nodes are typically lamp posts or
                                        service points that fall outside any building footprint.
dist_to_building        Distance in metres from the service point to the matched building.
                        Null for overlap matches and unmatched nodes.
activity_type           Land-use classification of the matched building from the buildings parquet
                        (e.g. "domestic", "non-domestic"). Null if unmatched.
